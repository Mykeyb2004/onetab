# 开发预览及发布打包

适用范围：本说明用于本地开发时快速预览 Manager 页面，以及发布前完成编译、检查和打包验证。

最后更新时间：2026-06-17

关联文件：

- `package.json`
- `preview.html`
- `src/preview/manager-preview.tsx`
- `src/preview/runtime-shim.ts`
- `export20260524.spd`
- `dist/`

## 1. 安装依赖

首次拉取项目后，先安装依赖：

```bash
npm install
```

建议使用 Node.js 22 或更新版本，避免较旧运行时对 Vite、Vitest 或 Rolldown 的支持不完整。

## 2. 启动网页预览

如果只是查看 Manager 主界面和大部分 UI 交互，不需要每次都把插件重新加载到 Chrome 扩展管理页，可以直接启动网页预览：

```bash
npm run dev:preview
```

默认预览入口：

```text
http://127.0.0.1:5173/preview.html
```

如果 `5173` 端口已被占用，以终端里 Vite 输出的实际地址为准，入口路径仍然是：

```text
/preview.html
```

设置页也可以在同一套预览环境里直接打开：

```text
http://127.0.0.1:5173/options.html
```

如果端口变化，设置页入口路径仍然是：

```text
/options.html
```

## 3. 载入之前保存的数据

预览页启动后会自动读取仓库根目录下的保存数据文件：

```text
export20260524.spd
```

首次打开 `preview.html` 或 `options.html` 时，系统会把该文件导入到浏览器当前站点的 `localStorage` 中。之后刷新页面会继续使用 `localStorage` 里的预览数据，不会反复覆盖你在预览环境里做的修改。

如果需要重新从 `export20260524.spd` 载入初始数据，打开：

```text
http://127.0.0.1:5173/preview.html?reset=1
```

设置页对应的重置入口是：

```text
http://127.0.0.1:5173/options.html?reset=1
```

这个入口会先清空当前预览页使用的 `localStorage`，再重新导入 `export20260524.spd`。

## 4. 预览环境能验证什么

网页预览环境会模拟以下 Chrome API 的最小行为：

- `chrome.storage.local`
- `chrome.storage.onChanged`
- `chrome.runtime`
- `chrome.tabs`
- `chrome.windows`

适合验证：

- Manager 页面布局
- 设置页基础表单
- 分组列表和标签列表展示
- 搜索
- 分组管理
- 导入导出
- 回收站
- 密度偏好
- 侧边栏偏好

仍需在真实扩展环境验证：

- 真实标签收纳
- 关闭浏览器标签页
- 恢复窗口或标签
- 右键菜单
- 快捷键
- 通知
- Manifest 权限
- Service worker 生命周期

## 5. 发布前检查命令

发布前建议按以下顺序执行：

```bash
npm run lint
npm run typecheck
npm run test
npm run build
```

如果本次修改触达真实扩展主流程，还应执行端到端测试：

```bash
npm run test:e2e
```

若要执行真实扩展浏览器 E2E，使用：

```bash
RUN_EXTENSION_E2E=1 npm run test:e2e
```

## 6. 编译与打包产物

正式编译命令：

```bash
npm run build
```

编译完成后，扩展产物会输出到：

```text
dist/
```

发布或人工验收前，至少确认：

- `dist/manifest.json` 存在
- `dist/` 能被 Chrome 作为已解压扩展加载
- Popup、Manager、Options、New Tab 和后台 Service worker 无启动错误

## 7. 在 Chrome 中加载打包产物

1. 执行 `npm run build`
2. 打开 Chrome 扩展管理页：`chrome://extensions/`
3. 开启“开发者模式”
4. 点击“加载已解压的扩展程序”
5. 选择本仓库下的 `dist/` 目录
6. 加载后手动检查 Popup、Manager、Options 和关键收纳/恢复流程

## 8. 设置界面入口

设置界面对应的扩展页面是：

```text
options.html
```

在真实扩展环境中可以通过以下方式打开：

- 在 Manager 页面顶部点击“选项”
- 在 Chrome 扩展管理页 `chrome://extensions/` 中进入 TabVault 详情，再打开扩展选项
- 直接访问扩展内地址：

```text
chrome-extension://<扩展ID>/options.html
```

编译后设置页文件位于：

```text
dist/options.html
```

注意：`preview.html` 和 `options.html` 在开发服务器下都会自动载入同一份 `export20260524.spd` 预览数据；如果想重新导入，直接打开各自的 `?reset=1` 版本即可。
