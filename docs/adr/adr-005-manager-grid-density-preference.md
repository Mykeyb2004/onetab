# ADR-005 Manager Grid Density Preference

- Status: Proposed
- Date: 2026-05-24

## Context

Manager 页右侧标签区将从单列列表改为响应式宫格视图。该变更需要在“信息密度是否更高”与“用户偏好是否稳定”之间做取舍。

如果完全依赖自动响应式切换，用户无法稳定表达自己更喜欢 `简洁` 还是 `增强` 信息密度；如果只依赖手动切换，又可能在窄宽度下出现增强卡片不可读的问题。

同时，本次变更需要扩展 `ExtensionSettings`，因此属于需要明确记录的数据结构决策。

## Decision

- Manager 右侧标签区只保留一种主视图：响应式宫格。
- 视图切换在产品层面定义为信息密度切换：`简洁` 与 `增强`。
- 在 `ExtensionSettings` 中新增一个偏好字段，用于记录用户选择的宫格密度。
- 当右侧可用宽度不足以承载增强卡片时，界面临时自动降为简洁模式。
- 自动降级状态只存在于运行时，不写回持久化设置；宽度恢复后，界面自动回到用户原偏好。

## Consequences

- 用户可以稳定保留自己的密度偏好，同时在窄宽度下获得安全降级。
- settings schema 需要新增 migration 与默认值补齐逻辑。
- manager UI 需要同时处理“用户偏好密度”和“当前生效密度”两个概念。
- 相关数据模型文档、测试和交互说明都需要同步更新。
